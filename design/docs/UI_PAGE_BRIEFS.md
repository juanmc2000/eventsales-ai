# UI Page Briefs

## 01 Dashboard / Home
Purpose: show portfolio performance and urgent commercial actions.

Must include:
- KPI cards: New Enquiries, Proposals Sent, Conversion Rate, Revenue Won
- Enquiry funnel
- Revenue trend
- Recent enquiries
- Top performing venues
- Enquiries by source
- AI suggestions

Tone: premium, lively, commercial, hospitality-focused.

## 02 Enquiry Details
Purpose: let a sales manager understand and act on an enquiry quickly.

Must include:
- enquiry headline and status badges
- booking overview
- AI summary
- confidence score
- missing information checklist
- recommended next step
- room/venue recommendations
- communication/activity tabs
- action button: Generate Proposal / Send Proposal

## 03 Proposals
Purpose: manage sent, viewed and draft proposals.

Must include:
- venue filter
- search
- proposal table
- status pills
- sent timestamp
- amount
- row actions
- new proposal button

## 04 Pricing Rules
Purpose: configure minimum spend logic and demand adjustments.

Must include:
- rule set tabs
- active rule table
- priority
- status
- last updated
- performance summary card
- create rule button

Rules covered:
- capacity-based rules
- time decay
- external demand signals
- manual adjustments

## 05 Personas
Purpose: manage AI communication behaviour.

Must include:
- persona cards
- persona scope: global / group / restaurant
- behavioural summary
- active status
- create persona action
- clone/edit actions

Required default personas:
- Corporate Persona
- Social Persona
- Ultra Luxury Persona
- Agency Persona

## 06 Workflows
Purpose: configure automated enquiry-to-deposit and follow-up workflows.

Must include:
- visual workflow canvas
- trigger node
- AI analysis node
- decision node
- customer communication actions
- proposal/deposit actions
- follow-up branches

Must support:
- wait steps
- branching logic
- cancellation conditions
- natural language workflow creation

## 07 Insights / Analytics
Purpose: show commercial and operational intelligence.

Must include:
- total enquiries
- conversion rate
- revenue
- average lead time
- revenue over time
- top venues
- performance tabs

## 08 Admin Environments
Purpose: internal technical admin console.

Must include:
- environment table: Production, Staging, QA, Development
- health status
- version
- last deployed
- deployed by
- environment details panel
- logs button

Style: utilitarian, dark, technical.

## 09 Admin User Management
Purpose: internal user creation and access management. Not client-facing.

Must include:
- user search
- role filter
- user table
- name, email, role, status, last login
- edit action
- create user action

Roles:
- Sales Director
- Sales Manager
- Sales Representative
- Read Only
- Administrator

Style: utilitarian, access-control focused, dark admin shell.
