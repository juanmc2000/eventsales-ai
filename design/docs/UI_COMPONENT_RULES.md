# UI Component Rules

## Non-Negotiable Application Shell
Every client-facing page must use:
1. Dark left sidebar
2. Dark top banner
3. Light main workspace
4. Rounded white cards
5. Purple / pink / orange / teal accent system

Admin pages may use a darker utilitarian shell, but must remain visibly part of EventSales AI.

## Sidebar
Required sections:
- Main: Home, Enquiries, Calendar, Proposals, Deposits / Bookings
- Configuration: Restaurants, Rooms, Pricing Rules, Personas, Workflows
- Insights: Dashboard, Reports, Performance
- Admin-only: Environments, Integrations, Users, Roles, Audit Logs, System Health, Deployment

Rules:
- Active item uses gradient or solid purple highlight.
- Icons are simple line icons.
- Bottom-left venue / tenant switcher for client pages.
- Admin shell uses darker, more technical styling and no venue photography.

## Topbar
Required elements:
- Global search input
- Date range selector where relevant
- Notifications
- Help icon
- User profile menu

Global search placeholder:
`Search enquiries, customers, venues...`

## KPI Cards
Each KPI card must include:
- Accent icon tile
- Metric label
- Primary number
- Delta indicator
- Tiny sparkline where useful

Allowed accent sequence:
1. Purple
2. Pink
3. Orange
4. Teal

## Tables
Tables must be clean and enterprise-grade.
Required:
- Compact row height
- Status pills
- Row action icon
- Search/filter controls above table
- Empty state
- Loading state

## AI Suggestion Cards
AI suggestions must include:
- Recommendation headline
- Explanation / rationale
- Confidence indicator where relevant
- Clear next action
- Optional impact tag

Never show AI recommendations without rationale.

## Workflow Builder
Must use a canvas-style flow diagram with:
- Trigger node
- AI analysis node
- Decision node
- Action nodes
- Success/completion node

Nodes should be readable and calm, not overly playful.

## Persona Cards
Persona cards should include:
- Abstract gradient header art
- Persona name
- Scope: Global / Group / Restaurant
- Short behavioural summary
- Active/inactive status
- Clone/edit actions

## Forms
Rules:
- Label above input
- Helper text below where needed
- Required fields marked clearly
- Group fields into cards
- Use right-hand contextual preview where useful

## Status Pills
Suggested status language:
- New
- Hot Lead
- Information Requested
- Proposal Sent
- Viewed
- Deposit Link Sent
- Deposit Received
- Closed Won
- Closed Lost
- Escalated
- Active
- Inactive
- Healthy
- Warning
- Error

## Admin Pages
Admin pages must be more utilitarian:
- Darker background
- Denser tables
- Fewer gradients
- Technical labels
- Auditability over visual flourish
- No client-facing hospitality imagery
